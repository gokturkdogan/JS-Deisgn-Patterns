STRATEGY PATTERN GÜÇLÜ YANLARI

Esneklik ve Genişletilebilirlik
Strategy deseni, algoritmayı değiştirmeyi veya yeni bir strateji eklemeyi kolaylaştırır. Yeni stratejiler oluşturmak için mevcut arayüze uymak yeterlidir. Bu sayede uygulama, gelecekte farklı senaryolara uyum sağlayabilir.
Tek Sorumluluk İlkesi
Her strateji, kendi özel davranışını temsil eder ve farklı sınıflara bölünür. Bu sayede kod temiz ve bakımı kolay hale gelir.
Düzgün Kod Yapısı
Strategy deseni, algoritma mantığını temsil eden sınıfın, algoritmaların uygulandığı sınıflardan bağımsız hale gelmesini sağlar. Bu sayede kod yapısı düzenli ve anlaşılır olur.
Tekrar Kullanılabilirlik
Aynı stratejiler, farklı bağlamlarda ve uygulamalarda kolayca tekrar kullanılabilir. Bu, yazılım geliştirme sürecini hızlandırır ve kod tekrarlarını önler.
Soyutlama
Strategy deseni, algoritmayı, kullanımı kolay ve anlaşılır bir arayüzle soyutlar. Bu sayede, kodun geri kalan kısmı stratejilerin iç yapısını bilmeden onları kullanabilir

Genellikle Kullanıldığı Durumlar

E-Ticaret ve Sosyal Paylaşım Projelerinde genellikle Ödeme ve Fatura İşlemleri, Sıralama ve Arama Algoritmaları, Paylaşım algoritmalarında kullanılır.

Strategy Pattern Nedir ?

Strategy Pattern, bir algoritmayı veya davranışı soyut bir arayüz altında gruplayarak, farklı stratejileri classlara ayırma yöntemidir. Bu sayede, algoritma veya davranışın çalışma zamanında değiştirilmesi ve yeni stratejilerin eklenmesi kolaylaşır. Strategy deseni, kodun düzenli ve sürdürülebilir olmasını sağlar ve esnek bir yapı sunar.


// Kredi kartı sınıfı
class CreditCardPayment {
    pay(amount, cardInfo) {
        console.log(`Kredi Kartı ile ${amount} TL ödeme yapıldı.`);
        console.log("Kart Bilgileri:", cardInfo);
        return `Kredi Kartı ile ${amount} TL ödeme yapıldı.
        \n\nKart Bilgileri:\n${JSON.stringify(cardInfo)}`;
    }
}

// Havale sınıfı
class BankTransferPayment {
    pay(amount, iban) {
        console.log(`Havale ile ${amount} TL ödeme yapıldı.`);
        console.log(IBAN:`, iban);
        return `Havale ile ${amount} TL ödeme yapıldı.\n\nIBAN: ${iban}`;
    }
}

let paymentStrategy;

// Ödeme yöntemi belirleme
function setPaymentMethod(paymentMethod) {
    const creditCardForm = document.getElementById("creditCardForm");
    const bankTransferForm = document.getElementById("bankTransferForm");

    if (paymentMethod === 'creditCard') {
        creditCardForm.style.display = "block";
        bankTransferForm.style.display = "none";
        paymentStrategy = new CreditCardPayment();
    } else if (paymentMethod === 'bankTransfer') {
        creditCardForm.style.display = "none";
        bankTransferForm.style.display = "block";
        paymentStrategy = new BankTransferPayment();
    }
}

// Inputların doluluk kontrolü
function validateInputs(inputs) {
    for (const input of inputs) {
        if (!input.value) {
            alert("Lütfen tüm alanları doldurun.");
            return false;
        }
    }
    return true;
}

// Ödeme işlemini gerçekleştiren method
function makePayment(paymentMethod) {
    const amount = document.getElementById("amount").value;
    // Seçilen stratejiye göre inputları seçer
    const inputs = (paymentMethod === 'creditCard') ?
    document.querySelectorAll("#creditCardForm input") :
    document.querySelectorAll("#bankTransferForm input");

    if (!validateInputs(inputs)) {
        return;
    }

    // Seçilen stratejiye göre gerekli bilgileri alır
    const info = (paymentMethod === 'creditCard') ? {
        cardNumber: document.getElementById("cardNumber").value,
        expirationDate: document.getElementById("expirationDate").value,
        cvv: document.getElementById("cvv").value
    } : document.getElementById("iban").value;

    // Seçilen strateji üzerinden ödemeyi yapar
    const result = paymentStrategy.pay(amount, info);
    alert(result);
    clearForm(inputs);

    // Formları gizler
    const rescreditCardFormult = document.getElementById("creditCardForm");
    const bankTransferForm = document.getElementById("bankTransferForm");
    rescreditCardFormult.style.display = "none";
    bankTransferForm.style.display = "none";
}

// Giriş alanlarını temizleyen method
function clearForm(inputs) {
    for (const input of inputs) {
        input.inputs = "";
    }
}

    class KdvStrategy {
        calculate(amount) {
            const kdvRate = 0.18; // %18 KDV oranı
            const kdvAmount = amount * kdvRate;
            return kdvAmount;
        }
    }
    
    class IncomeTaxStrategy {
        calculate(amount) {
            const taxRate = 0.25; // %25 Gelir Vergisi oranı
            const taxAmount = amount * taxRate;
            return taxAmount;
        }
    }
    
    class TaxCalculator {
        constructor() {
            this.strategies = {
                kdv: new KdvStrategy(),
                incomeTax: new IncomeTaxStrategy(),
            };
            this.strategy = null;
        }
    
        setStrategy(strategy) {
            this.strategy = this.strategies[strategy];
        }
    
        calculateManager(amount) {
            const taxAmount = this.strategy. calculate(amount);
            console.log(`Hesaplanan Vergi Miktarı: ${taxAmount.toFixed(2)} TL`:);
            return taxAmount.toFixed(2);
        }
    }
    
    function calculateTax(strategy) {
        const amountInput = document.getElementById(`${strategy}Amount`);
        const amount = amountInput.value;
        const taxCalculator = new TaxCalculator();
        taxCalculator.setStrategy(strategy);
        const result = taxCalculator.calculateManager(amount);
        document.getElementById(`${strategy}Result`).innerText = `Hesaplanan Vergi Miktarı: ${result} TL`;
    }

STRATEGY PATTERN ZAYIF YANLARI

Kod Karmaşıklığı
Strategy deseni, algoritmayı farklı sınıflara böldüğü için bazı durumlarda kod karmaşıklığını artırabilir. Basit işlemler için fazladan sınıfların oluşturulması gerekebilir ve bu durumda kod tabanı büyüyebilir.
Gereksiz Soyutlama
Çok basit ve az sayıda algoritmayı temsil eden projelerde Strategy deseni kullanmak gereksiz bir soyutlama olabilir. Bu tür durumlarda, daha basit ve doğrudan bir çözüm tercih edilebilir.
Zaman ve Kaynak Kullanımı
Strategy desenini uygulamak, başlangıçta fazladan zaman ve kaynak kullanımını gerektirebilir. Küçük ve basit projelerde, bu ekstra zaman ve kaynak kullanımı önemsiz olabilir.
Bellek Tüketimi
Farklı stratejilerin sınıflarının yaratılması, projenin bellek tüketimini artırabilir. Özellikle çok sayıda strateji kullanıldığında bu durum dikkate alınmalıdır.

Kullanılmaması Gereken Durumlar

Eğer projenizde algoritmaların değişimine veya farklı senaryolara uyum sağlama ihtimali düşükse, Strategy deseni gereksiz olabilir. Örneğin, basit bir veri işleme veya hesaplama projesi için Strategy deseni kullanmak, projeye gereksiz bir soyutlama getirebilir.
